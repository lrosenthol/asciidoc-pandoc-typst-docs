== Some real world content

=== Support for https://cawg.io[CAWG] Assertions

JPEG Trust Part 1 documents usage of three of the CAWG assertions defined at https://cawg.io[].  These assertions include:

* Identity assertions
* Metadata assertions
* Training & Data Mining assertions

The C2PA specification does not define these assertions, but they are syntactically compatible with the specification but are not guaranteed to be processed by C2PA-only implementations. However, the C2PA recently issued an https://spec.c2pa.org/specifications/specifications/2.4/identity/identity.html[informative document] that recommends the use of CAWG Identity assertions for human and organizational identity information.

=== The C2PA Provenance Model

The C2PA Technical Specification defines a provenance model centred on three core artifacts [C2PA §2.3]:

**Assertion**:: A data structure representing a statement — either created by the signer or gathered at claim generation time — concerning an asset. Assertions cover areas such as asset creation, edit actions, capture device details, bindings to content, and metadata.

**Claim**:: A digitally signed, tamper-evident data structure that references a set of assertions and the information necessary to represent the content binding [C2PA §2.3.2].

**Claim Signature**:: The digital signature on the claim created using the private key owned by a signer [C2PA §2.3.3].

[NOTE]
====
The `digitalSourceType` field on an ingredient assertion cannot be combined with an `activeManifest` field — the presence of both constitutes a validation failure [C2PA §15.11.3.2]. Its purpose is specifically to characterise ingredients that lack their own provenance record, preserving the descriptive integrity of the overall provenance chain. This mechanism is particularly relevant in Generative AI workflows where a model may incorporate style-reference assets or training exemplars that predate this ecosystem.
====
